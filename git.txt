) git fetch origin

What it really does:
Downloads commits, branches, and tags from the remote (origin) into your local repo without touching your working files.

👉 It updates remote-tracking branches like origin/main, not your current branch.

Example 1
Your teammate pushed commits to main

You run:

git fetch origin
Now origin/main is updated, but your local main is unchanged
Example 2
You want to inspect changes before merging

You fetch, then:

git log origin/main
Safe preview without affecting your work
2) git reset --hard origin/main

(you wrote mains — that’s wrong unless your branch is actually named that)

What it really does:
Forcefully makes your current branch identical to origin/main
→ Deletes all local commits + changes

Example 1
You messed up your code badly

Want a clean slate from remote:

git reset --hard origin/main
Example 2
You pulled experimental junk and regret it
This nukes everything and restores remote state

⚠️ Brutal truth: This is destructive. If you didn’t back up your work, it’s gone.

3) git rm --cached backend/.env

What it really does:
Stops tracking the file in Git, but keeps it on your disk.

Example 1
You accidentally committed .env

Remove from Git but keep locally:

git rm --cached backend/.env
Example 2
You want .env ignored moving forward

After removing:

echo "backend/.env" >> .gitignore

⚠️ Important: This does NOT remove it from commit history.

4) git reset --hard HEAD~1

What it really does:
Moves your branch pointer back 1 commit and deletes that commit permanently (locally).

Example 1
You committed broken code

Undo last commit:

git reset --hard HEAD~1
Example 2
You committed secrets accidentally
Remove commit locally before pushing

⚠️ If already pushed → this alone is useless. You need force push.

5) git push origin main --force

What it really does:
Overwrites the remote branch with your local branch state.

Example 1
You rewrote history using reset/rebase

Need to update remote:

git push origin main --force
Example 2
You removed a bad commit locally
Force push to delete it remotely

⚠️ Reality check:

This can break your team’s repo
If others pulled old commits → chaos
6) git switch -c master origin/master

What it really does:
Creates a new local branch master tracking origin/master

Example 1
You cloned repo but don’t have master locally

Create it:

git switch -c master origin/master
Example 2
You want to work on legacy branch master
This sets it up correctly with tracking
7) git push origin --delete branch_name

What it really does:
Deletes a branch from the remote repository

Example 1
Feature branch merged

Clean up:

git push origin --delete feature-login
Example 2
You created wrong branch name
Remove it from remote
8) git pull --rebase origin develop

What it really does:
Fetches develop and replays your commits on top of it

Instead of merge → creates a clean linear history

Example 1

Your branch:

A---B (your commits)

Remote develop:

A---C---D

After rebase:

A---C---D---B
